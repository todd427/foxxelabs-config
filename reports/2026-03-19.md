### 3. Scéal (`todd427/sceal`)
**Multi-voice audiobook pipeline using StyleTTS2**

Scéal turns a plain RTF or TXT story file into a fully-produced, multi-voice MP3 — distinct consistent character voices, prosodic nuance, and quality control throughout. All processing is local on GPU; no cloud API costs.

**Stack:** Python · FastAPI · StyleTTS2 · ffmpeg
**Server:** port 7331
**Venv:** `../venvs/scEnv`

**Architecture:**
```
story.rtf / story.txt
       │
       ▼
  segment.py          ← RTF/TXT → speaker-attributed JSON script
       │
       ▼
  characters.py       ← character name extraction + normalisation
       │
       ▼
  textprep.py         ← text normalisation (abbreviations, edge cases)
       │
       ▼
  prosody.py          ← punctuation → prosody markers
       │                 (...→<beat>  —→<beat>  --→<beat>)
       ▼
  render.py           ← JSON script → per-segment WAV via TTS server
       │
       ▼
  quality.py          ← MOS scorer (F0 autocorrelation + spectral centroid + SNR)
       │                 QA flags: CLIPPED  NOISE  POOR_QUALITY
       ▼
  server/server.py    ← FastAPI TTS server, StyleTTS2 model, job queue
       │
       ▼
  ffmpeg              ← WAV segments → MP3
```

| Commit | Description |
|---|---|
| `fix` | Vocos silent output — output tensor is `[re, im]` not `[log_mag, phase]` ← critical fix |
| `fix` | F4 resonator, pre-emphasis shelf, corrected fScale/pitchRange mapping |
| `feat` | Save output as WAV button (↓ WAV alongside ▶ speak) |
| `feat` | WAV analyser → auto-apply sliders: record your own voice, auto-extracts F0/LPC/rate/breathiness |
| `fix` | Rec panel state-driven UX with explicit 3-step instructions |

**The Vocos fix** was the most technically significant change of the day for this repo. The BSC-LT Vocos model outputs a complex STFT representation directly — the output tensor is `[real_components, imaginary_components]` interleaved across frequency bins and time frames. The original implementation assumed `[log_magnitude, phase]` (the more common Vocos output format), which produced entirely silent audio on the neural path. The fix decodes the tensor correctly via `decodeVocosOut()` before calling `vIstft()`.

**Synthesis pipeline in `App.jsx`:**
- 44-phoneme CMU dictionary with letter-to-sound fallback
- 3-resonator IIR formant filter bank with Rosenberg glottal pulse
- F0 contour with pitch declination, breathiness, creakiness, jitter
- 8 gender×age base presets (male/female × child/teen/adult/elderly)
- 9 archetype deltas (neutral, gruff, gentle, nervous, authoritative, cheerful, weary, menacing, whispery)
- WAV analysis: autocorrelation-based F0, LPC formant estimation, syllable rate, noise floor
- Full ONNX inference bridge for Vocos: mel spectrogram computation, STFT, inverse STFT

**SQLite schema (12 tables):**
`projects`, `milestones`, `sessions`, `worklog`, `budget`, `api_registry`, `api_project_map`, `api_usage`, `deployments_cache`, `deploy_history`, `machine_heartbeats`, `agent_actions`.

**Two bugs diagnosed and fixed:**

**Bug 1 — Shared DB across test modules.**
`DB_PATH` was a module-level constant, evaluated once at Python import time. Once `db.py` landed in `sys.modules`, every subsequent test module pointed at the same SQLite file. The symptom: `test_list_projects_empty` returned a project left over from `test_db.py`'s `test_context_manager_commits`. The fix: replace the constant with `_db_path()` — a function that reads `os.environ["RIALU_DB"]` dynamically on every `get_connection()` call. The `conftest.py` `fresh_db` fixture sets `RIALU_DB` to a new `tmp_path` file before each test function, so each test gets an empty database.

**Bug 2 — Slug collision in worklog tests.**
`test_worklog.py` had an `autouse` setup fixture that ran before every test function and tried to `INSERT INTO projects` with slug `worklog-test`. After the first test, the slug already existed in the DB — the second test's setup raised `sqlite3.IntegrityError: UNIQUE constraint failed: projects.slug`. The fix: removed `autouse` and replaced with an explicit `project_id` fixture that only runs for tests that request it. Combined with `fresh_db`, each test now gets a clean DB and a cleanly-created project.

**Result: 42/42 tests passing.**
